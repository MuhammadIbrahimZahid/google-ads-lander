# 📄 V1 Documentation — Google Ads Lander + GA4 Conversion System

## 🧭 Project Overview

This project is a learning-based implementation of a **full conversion tracking funnel** using:

- Next.js (Landing Page)
- GA4 (Google Analytics 4)
- Google Ads (linked, but not actively running campaigns)
- Vercel (deployment)

The goal of V1 was to understand:

> How a user journey flows from landing page → conversion event → analytics tracking → Google Ads integration.

---

# 🏗️ Architecture (V1)

```text
User visits landing page
        ↓
Clicks CTA button
        ↓
Navigates to /thank-you
        ↓
GA4 tracks page_view + custom event
        ↓
generate_lead event fired
        ↓
GA4 Realtime confirms event
        ↓
Google Ads account linked to GA4
        ↓
Event available for import as conversion
```

```env
NEXT_PUBLIC_GA_MEASUREMENT_ID= "your measurement id"
```

> Only environment variables prefixed with `NEXT_PUBLIC_` are available in client-side code.

## Installation Method

- Google tag (gtag.js) injected in `app/layout.tsx`
- Loaded via Next.js `<Script />`

## Events Tracked

### 1. Automatic Events

- `page_view`
- `session_start`
- `user_engagement`
- `scroll`

### 2. Custom Events

#### `generate_lead`

Fired when user reaches `/thank-you`

```ts
window.gtag("event", "generate_lead", {
  lead_source: "landing_page",
});
```

# 🧠 Key Learning Outcomes (V1)

This project demonstrates:

### 1. Event-based analytics model (GA4)

Everything is an event, including page views and conversions.

### 2. Conversion design pattern

```
Landing Page → Action → Thank You Page → Event
```

### 3. Separation of systems

- GA4 = tracking layer
- Google Ads = optimization layer

### 4. Integration pipeline

GA4 events can be imported into Google Ads as conversions.
